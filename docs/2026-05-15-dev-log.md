# 2026-05-15 开发日志 — V1 UX 与流程头脑风暴

> 接续 [`content-model.md`](content-model.md)。content-model.md 沉淀的是数据模型；本文档沉淀的是从注册到聊天到录入到校准的**用户旅程设计**——按头脑风暴的顺序记录决策与理由。

---

## 0. 起点

content-model.md 已经把数据模型定下来：`language_item` 是原子，`item_group` 是组织实体，mastery 绑 item 不绑 group，Tag/pgvector 砍出 V1。

剩下的问题：**用户究竟怎么使用这套东西？** 从注册一路走到第一次 chat，再到 ingestion，每一步的 UX 长什么样？

---

## 1. 注册哲学：chat-first

**核心理念**：进去就能 chat。

反向倒推：
```
chat 需要 → learner
learner 创建需要 → 极少字段
注册需要 → 极少字段
```

**决策**：
- 注册保留 email + password
- 不做邮箱验证（V1 维持现状）
- `account.name` 自动占位（用 email username 之类），不要求用户主动填
- 没有 SMS / 微信登录（V1 维持现状）

---

## 2. 第一个 Learner 创建

**字段**（首次创建只问这三个）：
- `name`（必填）
- `age`（建议必填，影响 AI 语气）
- `cefr_level`（带"不确定"选项；后续靠校准模式补）

**AI 人设单独抽出**：
- 不要在创建 learner 时弹 AI 人设配置
- 默认 Tina + female + 内置 persona prompt
- 在 settings 里有"自定义 AI 伙伴"入口
- 每个 learner 配套自己的专属 AI 人设（家庭里两个孩子可以有不同的 Tina）

**首次创建完**：一个大 CTA "去聊天"——**不是 dashboard**。

---

## 3. "教学 Session" 的洞察

这是这次脑暴最重要的概念翻转。

**原本的设想**：
- 用户先去"录入教材"页录材料
- 然后回来开始 chat 练习
- 两个动作两个流程

**翻转后的设想**：
- **Ingestion 是 chat 里的一个动作，不是独立功能**
- 用户在 chat 里拍照 / 录音 / 上传文件
- 材料瞬间变成本次 session 的 scope
- 用户"自然享受到素材带来的 focus 感，也顺手录入了"

**后果**：
- Dashboard 没有"录入教材"按钮
- 设置里有"管理我的素材"（看历史 group）
- 教材积累是 chat 行为的副产品——**今天和孩子聊一次，今天的素材就攒下来了**

---

## 4. 默认 Scope = 校准模式（不是硬 A1）

**初版方案**：没有 group 时，scope 硬卡 CEFR A1。

**驳回理由**（用户提）：
- 太死板，A2/B1 的孩子会觉得 Tina 很傻
- A0 完全零基础的孩子又会被吓到
- "应该试探出用户水平，让用户能聊起来"

**确定方案：前 5 轮 = 隐式 placement test**

```
Turn 1: Tina 用 A1 难度问 "Hi! What's your name?"
Turn 2: 看孩子回复——
   - 一个词 "Sam"          → 继续 A1，节奏放慢
   - 短句 "I'm Sam, I'm 8"  → 试探 A1+
   - 主动多说               → 试探 A2
Turn 3-5: ±半级范围微调；锁定一个工作 level
   ↓
推断出的 level 写回 learner.cefr_level
```

详细实现见 §11。

## 5. Ingestion 入口：常驻工具栏 + 底部抽屉

**用户的担忧**：
- 跳页面（标准流程，但打断 chat）
- 弹窗（不打断，但弱化录入概念）
- 都不理想

**第三种方案**：

```
┌──────────────────────────────────────────┐
│  正在练习: [Kids Corner B1 U4 L2  ▾]     │ ← 顶部 scope banner
├──────────────────────────────────────────┤
│  Tina: Hi! What colors do you like?      │
│  Sam:  I like red and blue.              │
│                                           │
├──────────────────────────────────────────┤
│  [📷] [🎤] [📎]   输入...        [发送]   │ ← 常驻三个入口
└──────────────────────────────────────────┘
```

**点 📷 → 底部弹 1/3 屏抽屉**：
- 不遮挡上面的对话（视野下方）
- 不导航走（不是新页面）
- 不是 modal（不阻塞）

**移动端**：bottom sheet
**桌面端**：右侧 panel
**同一心智模型**

---

## 6. Ingestion 抽屉：两层渐进披露

**核心原则**：默认相信 AI，编辑是退路。

### 第一层（80% 用户停这里）

```
┌─────────────────────────────────────────┐
│  ✓ 找到 14 个学习点                       │
│                                          │
│  📕 看起来是: Kids Corner Book 1 - Unit 4│
│      Colors and Clothing                 │
│                                          │
│  Words 10 · Patterns 2 · Phrases 2       │
│  其中 2 个已掌握，12 个 新内容            │
│                                          │
│  [ 仔细看看 ]            [ 用起来吧 →]   │
└─────────────────────────────────────────┘
```

"用起来吧" → 一键保存 + 关抽屉 + chat banner 切到新 group + Tina 接 "好！我们来聊聊这些 colors!"

### 第二层（剩 20% 想精细控制）

完整 items 列表（每行可改可删）+ AI 自标低置信度项加 ⚠️ + 保存目标 radio（AI 推荐项默认选中）+ "保存后 chat 将聚焦这 14 个学习点" 说明。

### 保存后的反馈

**绝对不弹"成功"toast**。直接让 Tina 用新 scope 接一句话。比如：

```
[scope banner 更新]

Tina: 好！我看到你今天学了颜色。
       What's your favorite color?
```

这一句话同时完成三件事：确认录入、演示新 scope、回到 chat 流。

### 保存动作时机

**抽屉里"保存" = 立刻写库**。简单粗暴。如果用户后悔提供"撤销刚才的录入" 30 秒内可用即可。不维护 session 内 pending state。

## 7. AI 提取的 Prompt 设计

**决策：V1 单次调用搞定**（OCR + 分类 + CEFR 打标 + 元数据）。理由：
- 拆 4 次调用延迟翻 4 倍，用户已经在等
- Doubao Vision 单次 3-5 秒能输出完整结构化结果
- 拆开后跨任务一致性反而差（OCR 错误传染分类）

### System Prompt 骨架

完整 prompt 见 content-model.md 后续会补的 §10。要点：
- 严格 JSON 输出
- 仅英语 items
- type ∈ {word, phrase, pattern}；pattern 必带 anchor
- CEFR 标签 + POS 标签 + confidence 标签
- 元数据：book_name / unit / lesson / page
- 跳过页码、版权、导航文字
- 不确定的字段 null + 加 warning，不要瞎猜

### 输出 schema

```json
{
  "source_type": "textbook_page" | "worksheet" | "handwritten" | 
                 "flashcards" | "screenshot" | "other",
  "metadata": {
    "book_name": string | null,
    "unit": string | null,
    "lesson": string | null,
    "page": string | null,
    "confidence": "high" | "medium" | "low"
  },
  "items": [
    {
      "text": string,
      "type": "word" | "phrase" | "pattern",
      "anchor": string | null,
      "cefr": "A1"..."C2" | null,
      "pos": "noun" | "verb" | ... | null,
      "confidence": "high" | "medium" | "low",
      "note": string | null
    }
  ],
  "warnings": [string]
}
```

### 健壮性

| 失败模式 | 处理 |
|---|---|
| JSON 解析失败 | 重试一次，prompt 加错误提示；再失败给用户友好报错 |
| items 为空 | 不是错误，UI："我没找到要学的内容，重拍 / 描述一下？" |
| 混了中文 | 后端正则过滤（`^[\x00-\x7F\s'-.,?!]+$`） |
| 图片完全无关 | `source_type: "other"` + 空 items |
| 多图重复 | 后端按 (type, text) 去重 |
| AI 把页脚当 item | confidence: low，UI 顶到列表前面让用户看 |
| 幻觉 | V1 不防，靠用户在抽屉里删；V2 加 evidence 字段 |

### 图像预处理

- 前端 resize 长边 1024px
- 统一转 JPEG quality 85
- 单次最多 5 张图

### Adapter 协议扩展

`LLMAdapter` 加一个新方法：

```python
class LLMAdapter(Protocol):
    async def invoke(self, messages, **kwargs) -> LLMResponse: ...
    async def invoke_vision(self, messages, **kwargs) -> LLMResponse: ...
    def stream(self, ...) -> AsyncIterator[str]: ...
```

不支持 vision 的 adapter 抛 NotImplementedError。`config.toml` 加 `[adapter] vision_provider` 单独配（STT/LLM/TTS/Vision 可能不同家）。

text-only 输入走 `invoke()` 而不是 `invoke_vision()`，prompt 共享，只是没有 image content block。

## 8. Tina 的主动 nudge

Tina 是产品最好的 onboarding 助手——在对话里教用户用产品，比 tooltip 强百倍。System prompt 里写规则：

- session ≥ 20 轮 + 用词重复 → "我们聊了不少了，要不要换个话题？"
- 孩子说"今天老师教了 X" / "I learned new words" → "Want to show me 📷?"
- session 结束（max_turns 触发软限）→ "今天聊得不错！要不要把刚才聊的存为合集？"

## 9. Session 概念对用户不可见

**Learner 主页 = "小明的练习"**：

```
小明的练习

📕  Kids Corner B1 U4 L2          昨天 · 12 轮
📗  自由练习                       3 天前 · 8 轮
🔖  机场词汇（生词本）              一周前 · 15 轮

[ + 开始新的练习 ]
   ↳ 选教材 / 选生词本 / 自由聊 / 上传新素材
```

- 用户看到的：上次聊到哪了 + 开始新的一次
- 后端的：session 表（计费、context window、统计单位）
- **"session" 是工程词，永远不让用户读到**
- 切换教材 / 切换 session / 开始新聊 = 合并为同一个动作"选择今天要聊什么"

## 11. 校准模式实现细节

### 状态放在 Learner 上

```python
class Learner(Base, TimestampMixin):
    # ... existing ...
    cefr_level: Mapped[str | None]  # NULL=未校准；A1..C2=工作 level
```

`NULL` 即"未校准"，足够表达，不需要单独 calibration_state 字段。

### Scope Computer 分支

```python
if learner.cefr_level is None:
    return scope_for_calibration()  # 不带词表，靠 prompt 控制
elif session.group_id is not None:
    return scope_for_group(session.group_id)
else:
    return scope_for_free_practice(learner.cefr_level)
```

### 校准 System Prompt

```
You are talking to a young English learner for the FIRST time.
You don't know their level yet. Goal: chat naturally while observing,
WITHOUT letting them feel tested.

Behavior:
- Open with simplest A1 greeting: "Hi! What's your name?"
- Sentences SHORT (5-8 words max)
- Adapt to response:
    single-word reply  → stay A1, slow down
    short full sentence → try A1+ (one new word)
    rich sentence       → try A2 vocabulary

DO NOT:
- ask "what's your level?"
- use words above B1 in first 5 turns
- lecture, correct, or quiz

Make them feel safe. First goal: they speak comfortably.
```

注意此 prompt **不带词表约束**——校准期没有 scope，靠 LLM 的 CEFR 知识自控。

### 每轮 Level 推断（并发后台调用）

主回路给孩子的回复不变，**并发**起一个轻量 LLM 调用分析这轮：

```python
chat_response = await llm.invoke(chat_messages)
return_to_user(chat_response)

asyncio.create_task(estimate_level_for_turn(turn))
```

estimate prompt：

```
Analyze this single learner utterance and rate their English level:
"{learner_text}"

Output JSON:
{
  "estimated_level": "A1"..."C2",
  "confidence": "high" | "medium" | "low",
  "evidence": "short note"
}
```

text-only 短调用，< 500ms，成本可忽略。

### 临时表 learner_calibration_turn

```
learner_calibration_turn
  learner_id, session_id, turn_sequence
  estimated_level, confidence
  PK (learner_id, turn_sequence)
```

### 收敛逻辑

```python
async def maybe_settle(learner_id):
    estimates = await load_estimates(learner_id)
    
    # 早收敛：前 3 轮一致
    if len(estimates) >= 3 and \
       len({e.estimated_level for e in estimates[-3:]}) == 1:
        return finalize(learner_id, estimates[-1].estimated_level)
    
    # 标准收敛：第 5 轮取众数
    if len(estimates) >= 5:
        modal = Counter(e.estimated_level for e in estimates).most_common(1)[0][0]
        return finalize(learner_id, modal)
    
    return None
```

`finalize` 把 `cefr_level` 写到 Learner 表，下一轮起切到 free practice 模式。

### Tina 自己不告诉孩子

校准完后**Tina 不说**"我现在了解你的水平了"——破坏沉浸感。直接静默切到正常 free practice。

家长后台可以看到："小明初始水平估计为 A1（基于 5 轮对话），可手动调整。"

### 边界

| 场景 | 处理 |
|---|---|
| 前 5 轮孩子一句话不说 | 不收敛，cefr_level 留 NULL，下次 session 继续 |
| 中英混杂 | 估计器只看英语部分 |
| 校准期间用户上传了教材 | 切到 group 的 hard scope，校准中断；下次 free practice 时重新校准 |
| 估出 C1（明显错） | 首次校准最高写 B1；超过留 NULL 等更多数据 |
| 用户手填了 level | 直接写入，跳过校准 |
| 想重新校准 | 设置里 "重新校准" → cefr_level 重置为 NULL |

### 一个值得注意的细节

校准估计器**只看孩子的发言**，不看 Tina 的发言。我们要测的是孩子的产出复杂度，不是 AI 用了多少词。

## 12. 顺序总览（从注册到第一次有效 chat）

```
1. /register
   email + password；account.name 自动占位
   ↓
2. 首登 → 强制 onboarding：建第一个 Learner
   name + age + cefr_level（含"不确定"）
   AI 人设默认 Tina + 默认 persona prompt
   底部小字 "稍后可在设置自定义 AI 伙伴"
   ↓
3. 创建完 → 大 CTA "去聊天"（不是 dashboard）
   ↓
4. 首次 chat (cefr_level=NULL → 校准模式)
   Tina 主动 onboard：
     "Hi 我是 Tina，第一次见面！
      想给我看你的课本吗 📷？
      或者直接说 hello！"
   底部三图标 📷🎤📎 常驻
   ↓
5. 孩子任选：
   - 直接说话 → 校准模式继续
   - 拍照 / 录音 / 上传 → 抽屉两层披露 → 保存 → 切 scope
   ↓
6. 5 轮内校准完成 → cefr_level 写回 learner
   静默切到 free practice 或继续在 group scope 中
```

**注册到产生第一个 turn，3-4 屏，≤ 2 分钟**。
