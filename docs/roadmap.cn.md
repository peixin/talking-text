## 阶段一：外部家庭（现在 → 约 2 周）

**待验证假设：「对陌生人家的孩子有效，而且他们第二周还回来。」**
"对自己孩子有效"带着创始人偏差——你在旁边、环境你装的、教材你拍的。下一圈验证需要
3–5 个不爱你的家庭。

两个硬门槛，然后邀请：

1. **部署上线。** 生产基建已经存在（2026-06-10 盘点）：远程 compose
   （`docker-compose-remote.yml`，接 `stillume-net`）、一键 `just docker-deploy`、
   共享 stillume-nginx 栈里的 `talking-text.stillume.com` vhost（上传体积 +
   SSE 不缓冲已修）。剩下是拧钥匙不是造车：填 `.env.deploy`、签证书、服务器上建
   `.env.app`（`SESSION_COOKIE_SECURE=true`）、执行部署。不做更多：不上 CI/CD、
   不上监控栈；stdout 的 `perf_logging` 就是全部可观测性预算。
2. **种子教材库**（critical path 第 4 步）。Follower 家长的首次体验必须是
   *选书 → 选单元 → 孩子开聊*——两分钟内出价值，不需要任何采集。优先录孩子学校
   真实在用的教材，Tot Talk 其次。
3. **邀请 3–5 个非创始人家庭**（熟人圈即可，这个规模邮箱登录够用）。

**及格线：** 5 家里 ≥3 家在第二周仍有 ≥2 次会话。同时观察：boundary check 的越界率
（每次会话的超纲词数）和 perf 日志里的各环节延迟。

**在此激活的义务：** 别人家孩子的录音落到我们服务器的那一刻，CLAUDE.md 里的隐私
自律条款从口号变成义务——定一条音频保留策略（如原始音频 30 天自动清理），所有存储
留在境内，落地时写进 dev log。

**邀请前加固——2026-06-11 已完成**（`2026-06-11-dev-log.md`）：系统提示词常驻
儿童安全规则（不当话题轻转移、不收集个人信息）+ 全部上传路径的可配置输入限制
（`config.toml [limits]`）。这是把产品放到陌生人孩子面前的前置条件。

**2026-06-11 状态——闸门已过，进入观察期。** 生产环境已上线；孩子真实在用的教材
已备好；已邀请同班同学家长（共同学习进度）。教材分享同日上线
（`learner-content-scope.cn.md` §6/§8.4 落地——私下链接/分享码，订阅/克隆/fork），
创始人维护的教材能触达每个受邀家庭并保持同步。本阶段剩余：**音频保留策略**，
以及盯住及格线（第二周留存、越界率、各环节延迟）。

## 阶段二：把产品论点做出来——掌握度 + stretch

**待验证假设：「看得见的进步是家长持续安排练习的理由」——以及边界外推论点本身。**
这是产品的灵魂，而它目前没有实现。可与阶段一并行。

设计与决策：[`phase2-mastery-stretch.cn.md`](phase2-mastery-stretch.cn.md)。
2026-06-10 已实施：

- ~~`learner_word_stats` 表~~——**设计阶段砍掉**（决策 A）：item 级
  `learner_item_stats` 早已存在，足以支撑 stretch 加权；周报在读取时从 turn 文本
  计算（规则 #3：派生，不物化）。
- Scope Computer V2 ✅——stretch = 下一单元词汇的 ~10%，按掌握度加权，会话种子
  轮换。「下一单元」排序靠 `item_group.position`（可空，自然排序兜底——本阶段
  唯一的 schema 变更）。
- 家长侧最朴素的呈现 ✅——**「孩子这周自己说出来的新词」**，纯列表，按
  stretch/课本/课外打标，落在家长工作台首页。同时检验论点（stretch 词出现在列表里）
  和留存钩子（家长为看它回来）。
- Follower 渐进解锁——**推迟**：会话开始时手选单元就是渐进解锁；自动推进等掌握度
  数据可信之后再做。

**及格线：** stretch 词在曝光后一周内真的出现在孩子口语里；家长主动提到这份报告。

## 阶段三：延迟，由数据决定

**待验证假设：「批式管线对创始人家庭以外的孩子太慢。」** 不要预先开工。阈值：真实
家庭使用中 p50 整轮延迟 > 约 4–5 秒，则先做**流式 TTS**（感知收益最大——回复音频
提前开播）；在 TTS 流式不再是瓶颈之前，批式 STT 和 LLM 都不动。日志撑不起这个
阈值，本阶段就保持空白。

## 明确现在不做的

这个仓库的历史失败模式是：核心未证先打磨周边。在上面的阶段闸门放行之前，以下都不做：

- **计费 / token 计量界面**——没有付费用户；注册表里的价格数据在那之前保持未消费。
- **PDF / MP3 教材导入**——文本 + 图片采集足以支撑种子库工作。
- **云端 `BlobStorage` 后端**——本地盘用到多设备或磁盘真正成为痛点（适配器缝已留好）。
- **整理工作台拖拽打磨、AI 归档建议器**——贡献者工具；目前贡献者只有一个人，他会点鼠标。
- **微信 / 短信登录**——受邀家庭邮箱够用；对外公开前与 PIPL 合规一起重提。
- **更多厂商适配器 / 模型注册表扩充**——抽象已完成；只在某个环节真要切换时才加条目。
