## 0. 我们在哪（2026-06-10）

核心闭环已经**用一个真实孩子验证通过**——孩子跟着手做的教材聊了真实会话，效果不错。
`content-lifecycle.md` §9 critical path 的第 1–3 步已完成：核心闭环验证、采集/沉淀
分离、整理工作台 V1。

但要对"验证了什么、没验证什么"保持精确：

| 已验证 | 尚未验证 |
|---|---|
| 孩子愿意和 Tina 进行范围内的英语对话，并且喜欢 | **产品论点本身**——掌握度驱动的 ~10% stretch（"把边界往外推一寸"）还是空壳：Scope V1 的 stretch 返回空，掌握度追踪没接进 scope |
| 采集 → 整理 → 练习的管线对运营者端到端可用 | **创始人不在场时的留存**——别人家的孩子，第二周 |
| 语音管线对至少一个家庭够用 | 没有创始人耐心加成的孩子对延迟的容忍度 |

下面每个阶段攻击当前最危险的假设。想调整顺序，先说出一个更危险的假设。

## 阶段一：外部家庭（现在 → 约 2 周）

**待验证假设：「对陌生人家的孩子有效，而且他们第二周还回来。」**
"对自己孩子有效"带着创始人偏差——你在旁边、环境你装的、教材你拍的。下一圈验证需要
3–5 个不爱你的家庭。

两个硬门槛，然后邀请：

1. **部署上线。** compose 栈已经能跑（`docker-compose.yml` 全栈集成测试）——租一台
   国内轻量服务器，加域名 + HTTPS，跑起来。不做更多：不上 CI/CD、不上监控栈；
   stdout 的 `perf_logging` 就是全部可观测性预算。
2. **种子教材库**（critical path 第 4 步）。Follower 家长的首次体验必须是
   *选书 → 选单元 → 孩子开聊*——两分钟内出价值，不需要任何采集。优先录孩子学校
   真实在用的教材，Tot Talk 其次。
3. **邀请 3–5 个非创始人家庭**（熟人圈即可，这个规模邮箱登录够用）。

**及格线：** 5 家里 ≥3 家在第二周仍有 ≥2 次会话。同时观察：boundary check 的越界率
（每次会话的超纲词数）和 perf 日志里的各环节延迟。

**在此激活的义务：** 别人家孩子的录音落到我们服务器的那一刻，CLAUDE.md 里的隐私
自律条款从口号变成义务——定一条音频保留策略（如原始音频 30 天自动清理），所有存储
留在境内，落地时写进 dev log。

## 阶段二：把产品论点做出来——掌握度 + stretch

**待验证假设：「看得见的进步是家长持续安排练习的理由」——以及边界外推论点本身。**
这是产品的灵魂，而它目前没有实现。可与阶段一并行。

- `learner_word_stats` 表——每轮增量 upsert，历史从 `turn.text_user/text_ai` 回填
  （规则 #3：从 turn 文本派生，不建事件表）。**动 schema 前需先确认设计。**
- Scope Computer V2：stretch = 下一单元词汇的 ~10%，按掌握度加权。
- 家长侧最朴素的呈现：**「孩子这周自己说出来的新词」**——纯列表，不做图表。这一个
  产物同时检验论点（stretch 词出现在列表里）和留存钩子（家长为看它回来）。
- Follower 渐进解锁（critical path 第 5 步；与 `learner-content-scope.md` 的订阅
  机制重叠）。

**及格线：** stretch 词在曝光后一周内真的出现在孩子口语里；家长主动提到这份报告。

## 阶段三：延迟，由数据决定

**待验证假设：「批式管线对创始人家庭以外的孩子太慢。」** 不要预先开工。阈值：真实
家庭使用中 p50 整轮延迟 > 约 4–5 秒，则先做**流式 TTS**（感知收益最大——回复音频
提前开播）；在 TTS 流式不再是瓶颈之前，批式 STT 和 LLM 都不动。日志撑不起这个
阈值，本阶段就保持空白。

## 明确现在不做的

这个仓库的历史失败模式是：核心未证先打磨周边。在上面的阶段闸门放行之前，以下都不做：

- **计费 / token 计量界面**——没有付费用户；注册表里的价格数据在那之前保持未消费。
- **PDF / MP3 教材导入**——文本 + 图片采集足以支撑种子库工作。
- **云端 `BlobStorage` 后端**——本地盘用到多设备或磁盘真正成为痛点（适配器缝已留好）。
- **整理工作台拖拽打磨、AI 归档建议器**——贡献者工具；目前贡献者只有一个人，他会点鼠标。
- **微信 / 短信登录**——受邀家庭邮箱够用；对外公开前与 PIPL 合规一起重提。
- **更多厂商适配器 / 模型注册表扩充**——抽象已完成；只在某个环节真要切换时才加条目。
