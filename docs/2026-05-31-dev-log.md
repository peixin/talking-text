# 开发日志 / Dev Log (2026-05-31)

本日志记录了今天完成的教材父子层级体系（Parent-Child Textbook Hierarchy）、词句管理页面拆分、智能练习范围边界限制（Smart Practice Scope Bounding）、动态详情页布局打磨，以及全链路交互式面包屑路径（Tag Path Breadcrumbs）的完整重构与优化工作。

## 1. 核心教材体系功能升级 / Parent-Child Hierarchy Upgrades

### 1.1 子孙节点词句数递归计算 / Recursive Descendant Items Count
- **功能描述**：为解决上级节点（如“教材/书名”、“册次”、“单元”）在目录中显示 `0 词句` 导致看起来“空空如也”的问题，我们实现了子孙节点学习点数量的深度递归合并计算。
- **防止死循环机制 (Loop Prevention)**：在树形遍历时，加入了严格的 `visited` Set 追踪保护，从根本上防止了数据库或内存结构中潜在循环引用导致的 CPU 挂起死锁。
- **展示优化**：
  - 在 `/parent/materials` 列表页和 `/parent/materials/[groupId]` 详情页中：
    - 若本身直属词句数为 0 但子级有词句，展示：`包含 X 个词句（含子级）` (Indigo 标识徽章)。
    - 若本身直属词句数不为 0 且子级也有，展示：`X 个词句（含子级共 Y 个）`。

### 1.2 词句管理页面“浏览与编辑模式”拆分 / Items View/Edit Split
- **只读浏览模式 (`/parent/materials/[groupId]/items`)**：
  - 默认进入只读的 **View Mode**，支持**递归提取该节点及其所有后代节点的所有词汇与句式**（通过给 `groups.get` API 传入 `recursive: true` 参数实现）。
  - 对非直属的词句，在名称右侧渲染一个精致的来源定位徽章：`来自: Lesson X`。点击徽章可直接免路由跳转并打开该具体子章节的独立编辑页面。
- **独立编辑模式 (`/parent/materials/[groupId]/items/edit`)**：
  - 传统直接管理和增删改词句的页面拆分到了独立的 **Edit Mode**，确保层级结构分明，数据流向清晰，避免越级修改的逻辑混乱。

### 1.3 智能练习范围与 AI 边界限制 / Smart Practice Scope & AI Bounded Context
- **上限阈值控制 (Context limit confirmation)**：如果选中的父级节点下递归包含的词句数**超过 100 个**，在点击“对话练习 (Start Dialogue)”时，前端会弹窗预警推荐用户收敛范围，但允许用户继续。
- **后端智能熟练度优先降噪与切片 (Mastery-Aware Scoring & Slice)**：
  - 当 scope items 数量超过 100 时，后端 `get_scope()` 自动查询当前学习者的 mastery 记录（`LearnerItemStats`）。
  - 根据熟练度对所有词句进行优先级评分排序：
    - **最高优先级 (Score: -1.0)**：未曾练习过的新词句（零熟练度记录）。
    - **次高优先级 (Score: correct_count)**：练习过但未熟练掌握的错题、盲区词句（正确次数越少越优先）。
    - **最低优先级 (Score: 1000.0 + correct_count)**：已掌握（`mastered_at` 不为空）的熟练词句。
  - 筛选排序后切片保留**前 100 个**最需要练习的词句输入给大模型，保障高交互性能和高度专注的针对性教学。

## 3. 全链路交互式面包屑路径重构 / Full-Depth Tag Path Breadcrumbs

为了实现用户对“我当前在哪一级别、我后面还有多少个级别，这本教材深度有多少”的直观掌控，我们重构了面包屑路径，建立了根节点到最深子孙节点的全路径追溯，并实现了独创的基于索引的差异化交互分裂（Index-based Interactive Splits）：

### 3.1 完整路径追溯算力 (Full-Branch Tracing)
- 实现了 `leafNode` 计算挂钩，自当前节点向下检索首条未归档的子孙链路至叶子节点。
- 从叶子节点反向逆推至根节点，生成全局一致的全路径层级列表（例如：`考试: IELTS › 题型: Speaking › 单元: Starter Unit 3 › 课次: Lesson 2`）。

### 3.2 四段索引差异化交互与精致样式布局 (Index-Based Splits)
- **1. 历史祖先节点 (`idx < currentGroupIdx`)**
  - **样式与指针**：精致白色实体背景卡片，带浅灰边框和 hover 投影，手型指针。
  - **交互**：为标准链接，点击能顺畅返回其对应的上级详情页。
- **2. 当前活动节点 (`idx === currentGroupIdx`)**
  - **样式与指针**：醒目的紫色主动高亮边框和卡片 (`border-indigo-500 bg-indigo-50/70`)，左侧带有指示小手 `👉`，默认指针。
  - **交互**：不可点击，防止重复刷新。
- **3. 直接子节点级 (`idx === currentGroupIdx + 1`)**
  - **样式与指针**：交互式蓝色虚线边框卡片，卡片值展示为明确的动作型占位文字 `[ 选择${levelTitle} ]`（如：`[ 选择单元 ]`），右侧配有上下弹跳的 `👇` 指示器。手型指针。
  - **交互**：
    - 在详情页中：点击立即触发 Snappy flat 客户端物理滚动（0 延迟平滑滚动至下方“子章节目录卡片”），并伴随子章节卡片的 1.5 秒紫色缩放脉冲高亮闪烁动效（彻底解决在长屏幕无需滚动时“点了没反应”的视觉疑惑）。
    - 在页面头部组件 (`TagPathHeader.tsx`) 中：点击直接路由链接至详情页并自动锚定 `#sub-chapters-list`，实现跨页锚定加载后闪烁引导。
- **4. 孙子及更深节点级 (`idx > currentGroupIdx + 1`)**
  - **样式与指针**：极简高级的精细白底边界卡片 (`border-slate-100 bg-white text-slate-500`)，卡片值展示为静态层级占位符 `[ ${levelTitle} ]`（如：`[ 课次 ]`）。正常指针 (`cursor-default`)。
  - **交互**：不带任何点击和路由效果，亦无任何禁用、锁止等负面暗示。用于帮助用户清晰感知未来全部后续层级名称与深度。
