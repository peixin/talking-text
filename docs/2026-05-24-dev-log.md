# 2026-05-24 开发日志 — 素材类型与父级选择器解耦

> 本次只动 `parent/materials` 这条线，目的是把「素材类型」和「所属父级」从
> 死板的枚举 / 单层下拉里解放出来：AI 推断 + 用户可自由输入 + 父级可内联新建。
>
> 不改数据库 schema，不改其他模块的行为。

---

## 0. 起点 — 用户视角的问题

进 `/parent/materials/<groupId>` 编辑素材时：

1. **「素材类型」是 6 选 1 的 `<select>`**（`textbook_book` / `textbook_unit` /
   `textbook_lesson` / `personal_collection` / `quick_practice` / `review_set`），
   想自己起个名（比如「夏令营听力包」「TOEFL 词汇」）做不到。
2. **「所属父级素材」是一个 flat 下拉**，只能选已有的素材，没法在这里直接新建一个
   作为父级；UX 给人「只能挂在一级下面」的错觉。
3. 实际上 DB 层 `item_group.parent_id` 是 self-FK，**树形深度本来就没限制**，
   `MaterialsClient` 的 `TreeViewNode` 也是递归渲染——是 UX 没把这层暴露。

---

## 1. 设计岔路：`kind` 怎么放开

`kind` 这一字段里其实**混了两种语义**：

| 语义     | 例子                                                   | 行为                                       |
| -------- | ------------------------------------------------------ | ------------------------------------------ |
| 层级标签 | `textbook_book` / `textbook_unit` / `textbook_lesson`  | 纯描述性，决定图标和颜色                   |
| 功能种类 | `quick_practice` / `review_set` / `personal_collection` | 代码里有硬编码，触发特定行为（如 chat 的随手练习抽屉） |

三个选项：

- **A. 完全自由 + 已知常量保留行为** ✅ 选这个
- B. 拆成两个字段（kind 留小枚举 + 新增 label）
- C. 保持 enum，但白名单移到 `config.toml`

B 最干净但要迁移；C 仍然死板。A 的代价是：代码里继续依赖 `"quick_practice"` 这样的
**字符串常量**，但 UI 不再强制选这几个值——这与 DB 层 `kind: String(30)` 早就允许
任意字符串的事实一致。

---

## 2. 父级选择器：Combobox + 内联新建

`<select>` 换成一个组合控件：

- 顶部触发按钮显示**层级路径**（`Book / Unit / Lesson`），而不只是叶子名
- 展开后是搜索框 + 过滤后的列表（按路径模糊匹配）
- 输入框里有内容、且没有 exact match 时，列表底部出现
  **`+ 新建『xxx』作为父级`**
- 点了立即调 `createGroup({ name, kind: "textbook_book", items: [] })`
  创建空容器并选中——孩子的保存动作不被打断

避免循环依赖：传入 `disabledIds`（self + descendants），picker 把它们灰掉。
descendants 计算抽成 `descendantIds()` 工具函数。

---

## 3. AI 推断的接口扩展

`POST /ingest/extract` 的 LLM 输出 schema 加一个字段：

```diff
  "metadata": {
    "book_name": string | null,
    "unit": string | null,
    "lesson": string | null,
    "page": string | null,
+   "kind_label": string | null,
    "parent_id": string | null,
    "cefr_level": "A1" | ... | "C2" | null,
    "confidence": "high" | "medium" | "low"
  }
```

Prompt 里新增第 4 条规则约束 LLM：

- `kind_label` 是 2–10 字的自然语言标签（中文或英文均可）
- **优先复用现有素材里已用过的 label**（一致性 > 创造性）
- 一次性 / 没有层级感的练习用 `"练习"` / `"Practice"` 就行

前端预览页直接把 `metadata.kind_label` 灌进 `KindInput`；如果 LLM 没给（旧响应或
低置信度），回退到老的「lesson 有就 lesson、unit 有就 unit、否则 book」推断。

## 5. 行为兼容性

- 现有 DB 里所有 `textbook_book` / `quick_practice` 等记录原样工作
- `IngestDrawerClient` 在 chat 侧创建即时练习时仍写死 `kind: "quick_practice"`——
  字符串常量行为不变
- 6 个已知 kind 在 UI 上仍按本地化 label 显示（chips / tree badge / 路径括号里）；
  用户输入「夏令营听力」就显示「夏令营听力」
- 父级 picker 现在按**层级路径**展示，孩子可任意嵌套深度（DB 早就支持，只是没暴露）

## 7. 已知遗留 / 下一步

- 内联新建父级时 kind 默认 `"textbook_book"`——能用但不够灵活。后续可以让 picker
  弹出小气泡也填 kind，或者新建后立刻跳转到详情页让用户继续编辑
- `KindInput` 的常用 chips 取自「内置常量 ∪ 账号已用过的 kind」，没做使用频次排序；
  数据多起来后值得加 ORDER BY count DESC
- zh-TW 翻译滞后于 zh-CN 和 en；不只是 MaterialPickers，整个 Materials / Ingest /
  Scope / Onboarding 都缺。一个独立的翻译收尾任务
- prompt 里要求 LLM「优先复用现有 label」的效果需要灰盒看几个真实样本，
  必要时再加 few-shot 示例
