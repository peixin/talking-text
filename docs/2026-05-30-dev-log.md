## 0. 起点 — 为什么"混乱"

教材系统把两件本该分开的事压成了一件：**采集碎片**（每天拍照/输入）与
**拼装成书**（Book→Unit→Lesson 树）。旧实现让每张照片在采集那一刻就"自动归位"
进树——AI 推断 `parent_id`、把整库塞进 prompt、按位置猜 `kind`、`levels` 数组模糊
并树。归档这件事本质模糊、易错、又跟核心循环无关，却被做成了自动主路径，于是三个
症状同时出现：层级太复杂、UI 乱、概念模型没想清。

确立的原则（详见 content-lifecycle.md §1）：

> **采集是 append-only 且"笨"的；结构化是另一个、刻意的动作，由极少数人（先是运营者）
> 完成，绝不强加到每张照片上。**

并区分了两类用户（跟随者 / 贡献者）与"渐进式"的两个含义（消费渐进 / 创作渐进）。

## 5. 自查发现的存量缺口（非本次回归，待后续）

未发现本次引入的 bug；以下两处是**沿袭旧 `levels` 分支的既有缺口**，建议后续补：

- **`tag_path` 路径无环检测**：非 `tag_path` 的 `parent_id` 分支有 `_check_cycle`，但 `tag_path`
  组装路径没有。实践中因 `_assemble_tag_path` 逐层从 root 精确匹配、很难构造出环，但理论上仍应补一道防护。

  不会触发锁校验（旧 levels 分支同样如此）。"在锁定树下新增内容"是否该受锁限制，需产品决定。

二者均与本次「采集/成书」拆分无关，且为既有行为，故本次未改，仅记录。

## 7. 整理工作台改版（袋级 + AI 预判）+ 家长台/教材页产品化

用户反馈：一个词一个词点选太累；应**以「一次录入（采集袋）」为整体**来整理，且让 **AI 给预判/default、做到 ~80%、人只微调**（用户很懒）。据此改：

**后端**（`544a618`）
- `GET /organize/inbox` 改为**袋级**：返回 `capture_bags`（每袋含其词），不再是扁平 `capture_items`；练习候选不变。
- `POST /organize/suggest-bag` —— LLM 根据「袋内词 + 现有 tag 路径」预判一个 `tag_path`（优先复用已有路径）；任何失败**兜底为 [袋名]**，所以不依赖 LLM 可用性。
- `POST /organize/file-bag` —— 把整袋的词原子移动到 `tag_path` 叶子（复用 `_assemble_tag_path`），并归档空袋。

**前端**（`544a618`）：organize 页改为**一袋一张卡**：词预览 + 一个**预填 AI 预判路径**的输入框 + 「整袋归位」一键。挂载时并发为每袋拉取建议（兜底袋名），人改路径文字即可。练习候选改为**一个路径输入 + 全部归位**（用 `createGroup(tag_path, items=候选)` 一次入库）。

**教材页**（`aa8a299`）：`/parent/materials` 把 **canonical tag 树**（教材结构）与**未整理采集袋**拆成两块分开展示（之前混在一棵树里，导致「看不到 tag 树」）；采集袋块带「去整理 →」入口；`useKindLabel` 给 `tag`/`generic` 补上「标签」标签（新增 `Materials.kind_tag`，三语）。

**家长台**（`d88d16e`）：`/parent` 从简陋占位重做成产品化 hub —— 当前学习者卡 + 「录入→整理→对话」三步主流程卡（带实时计数：N 本教材 / N 袋待整理高亮）+ 孩子卡片 + 次级入口。

验证：`just check` 全绿。**均未运行时验证**（由用户发起）。`suggest-bag` 的 LLM 调用为外部依赖，已做兜底，但实际预判质量需真人在真数据上观察。

## 8. 教材录入与内容建模升级（底稿解耦 + 胶囊可编辑层级 + 重新提取交互闭环）

我们在本阶段彻底落地了三项深水区的内容建模与录入闭环功能，将「采集」与「整理成书」的底层解耦（代号 B 升级）及生命周期交互彻底闭环：

### 改动细节

#### ① IngestionBatch (方案 B) 数据库彻底解耦与继承
- **数据库表**：在 `backend/app/storage/models/content.py` 中增加了 `IngestionBatch` 模型（含 `source_type`, `source_raw_text`, `media_urls`），在 `ItemGroup` 中增加了外键 `ingestion_batch_id` 与 `level_title` 字段。已生成并成功应用 Alembic 迁移脚本。
- **接口与继承**：
  - 新建 capture bag (`POST /groups`) 时即时创建并绑定 `IngestionBatch`。
  - 在整袋归位 (`POST /organize/file-bag`) 时，目标叶子节点将**自动继承** capture bag 的 `ingestion_batch_id` 引用并设置对应节点的 `level_title`。
  - 待整理收件箱 (`GET /organize/inbox`) 支持高性能批量联表拉取所有未整理袋子的 `IngestionBatch` 底稿原文返回给前端。

#### ② “以 Vision 启动，以纯文本微调” 在前台彻底闭环
- **录入抽屉 (`IngestDrawerClient.tsx`)**：
  - 增加 OCR transcribed 底稿状态 `rawText`，在预览界面下渲染高度精致的 details 结构**「底稿与二次微调 (AI OCR Draft)」折叠面板**。
  - 家长可以随时修改原始底稿，点击 **“重新提取 (Re-extract)”**。此时系统发起 pure-text 快速低成本提取并立刻刷新卡片中的单词列表，形成交互闭环。
- **待整理工作台 (`OrganizeWorkbenchClient.tsx`)**：
  - 同步引入 `bagRawTexts` 与 `extractingBags` 状态，在每张 capture bag 卡片底部绘制底稿微调区。
  - 家长在工作台就地编辑底稿点击“重新分析”时，工作台会调用新封装的 `extractIngestionAction` 获取新 items，然后通过 `updateGroupAction` 更新捕获袋数据并调用 `syncFromServer()`，卡片单词列表**原地无感刷新**，整个采集整理链条彻底闭合。

#### ③ 胶囊层级卡片支持 editable input titles 与 左右 Move 绑定
- **前台路径编辑器 (`OrganizeWorkbenchClient.tsx`)**：
  - 将 `paths` 类型由 `string[]` 升级为 `{ name: string; level_title: string }[]` 对象数组。
  - 将胶囊中原本只读的 `LEVEL_LABELS` 替换为 **inline editable text input**。家长可以双击自由定制每个级别的层级名（例如原本是写死的“单元”，可以直接编辑改为“课次”等）。
  - **排序位置绑定**：点击 `ArrowLeft` / `ArrowRight` 移动节点时，胶囊的标题（level_title）会和胶囊的值（name）作为对象**一起发生位置交换**，完美符合直觉，并在“整袋归位”时一并持久化提交给后端。
- **教材详情面包屑 (`GroupDetailClient.tsx`)**：
  - 开发了 `getLevelTitle` 追溯函数，动态读取并展示自定义过后的层级标题，完美替换了原先写死的Fallback标签。

我们在本阶段的最后对教材详情页面进行了深度的只读与编辑分离，并彻底解决了三项长期困扰的 Popover 交互与警告 Bug：

### 修复与重构细节

#### ① 教材详情页 View/Edit 模式彻底分离与 Edit 入口补齐
- **只读查看模式 (`readOnly={true}`)**：在 `/parent/materials/[groupId]` 详情页下，所有原本的 Input 输入框（包含教材名称、教学元数据、提示词补充等）均被重构为精美的只读 static 展示组件。去除了所有追加/删除按钮与删除 Danger Zone，且为 Tag Path 路径提供了专属的 read-only 静态胶囊视图。同时在悬浮工具栏中补齐了此前因 `git restore` 丢失的 **“编辑教材”** 跳转入口，引导用户安全进入编辑。
- **互动编辑模式 (`readOnly={false}`)**：在 `/parent/materials/[groupId]/edit` 路径下，渲染完整的层级与数据修改工具供用户深度微调。

#### ② Base UI PopoverTrigger Button 预期警告消除
- 在 `GroupDetailClient.tsx` 中，针对所有以 `div` 或 `input` 等非 `<button>` 元素包装的 `PopoverTrigger` 组件，均配置了 `nativeButton={false}` 属性，从而彻底消除了控制台频繁报出的 `Base UI: A component that acts as a button expected a native <button> because the nativeButton prop is true` 规范报错。

#### ③ 路径输入框点击导致 Popover 瞬间消失 Bug 根治
- 在层级属性与目录名输入框中，由于点击、悬浮等交互事件会冒泡至 `PopoverTrigger` 上，导致 Popover 在展开的一瞬间又被触发了 Close 行为。
- **事件传播拦截**：为两个 Autocomplete 输入框全部绑定了完整的冒泡拦截机制：
  ```typescript
  onClick={(e) => e.stopPropagation()}
  onPointerDown={(e) => e.stopPropagation()}
  onMouseDown={(e) => e.stopPropagation()}
  onTouchStart={(e) => e.stopPropagation()}
  ```
  该拦截机制完美根治了“点击输入框直接闪退/关闭弹窗”的现象。

#### ④ 删除/清空属性值被 Fallback 强行回弹重置 Bug 修复
- **根因诊断**：此前层级属性输入框 value 绑定为了带有 `||`（逻辑或）的 Fallback：
  ```typescript
  const currentTitle = levelTitles[idx] || LEVEL_PRESETS[idx] || `层级 ${idx + 1}`;
  ```
  当家长在页面上清空属性文字（例如把 “教材” 二字删空）时，状态机捕获到空字符串 `""`（属于 falsy 值），从而在渲染层强行触发 Fallback 回退显示 `"教材"`，导致无法删空或自主输入新词。
- **优雅解耦**：将绑定值精准重构为 state 的 `levelTitles[idx] ?? ""`。同时，把后备的默认退避标题移出 binding 层，完全下沉到 HTML 原生 `placeholder`（例如展示为半透明的占位文字 `“教材”`），使输入框既具备了自由清空、完全定制的能力，又保留了极佳的智能提示引导。

#### ⑤ 已整理/空采集袋在教材页展示过滤与一致性维护
- **业务表现**：当家长在 `/parent/organize` 点选“整袋归位”后，采集袋（`quick_practice` 组）中的所有单词会被安全移出并归档该袋子。此时，该袋子已成为完全清空的临时外壳。
- **发现的隐患**：
  1. 家长此前在 `/parent/materials` 底部的“已归档素材”列表中，仍会看到这些已整理空袋，极易引发误解并引发手动“恢复”操作。
  2. 一旦空袋被手动“恢复”为 active 状态，由于它已经没有单词（`item_count === 0`），它会在教材页顶部的“待整理采集”栏中出现（显示为 0 词），但点击“去整理”进入 `/parent/organize` 后却因为工作台本身过滤了空袋子而空空如也，造成两端显示不一致。
- **一致性治理**：
  1. 在 `MaterialsClient.tsx` 中，对 `archivedGroups` 增加了 `g.kind !== "quick_practice"` 过滤。已整理/已归档的采集袋将永远在底部归档列表中隐形，从源头上杜绝了由于手动恢复空袋子造成的混乱。
  2. 对教材页顶部的 `captureRoots` 计数增加了 `n.group.item_count > 0` 的拦截。只有当活跃采集袋中确实包含需要整理的重点词句时，才会展示在“待整理采集”条目中，与 `/parent/organize` 工作台过滤器和 `/parent` 仪表盘计数保持了完美一致。
